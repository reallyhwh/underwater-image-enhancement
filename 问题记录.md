## 问题1：WF-Diff显存不足

**发现时间**：2026-04-07

### 问题描述

运行WF-Diff测试LSUI数据集时，出现显存不足错误：

```
torch.OutOfMemoryError: CUDA out of memory.
Tried to allocate 1.41 GiB.
GPU 0 has a total capacity of 6.00 GiB of which 0 bytes is free.
```

### 原因分析

| 数据集 | 图像数量 | 图像尺寸 | 显存需求 |
|--------|----------|----------|----------|
| UIEB | 89张 | 256×256 | ~2GB（6GB够用） |
| LSUI | 400张 | 400×300 ~ 1280×1024 | ~4-8GB（6GB不够） |

Diffusion模型显存占用与图像尺寸的平方成正比。

### 解决方案

**动态Resize策略**：在推理前将大图像缩小到512×512，推理完成后再恢复原始尺寸。

修改文件：`WF-Diff-main/basicsr/models/Wfdiff_model.py`

```python
def test(self):
    # 记录原始尺寸
    target_size = 512
    original_h, original_w = self.original_size
    
    # 只对大于512的图像进行resize
    need_resize = original_h > target_size or original_w > target_size
    
    if need_resize:
        lq_resized = F.interpolate(self.lq, size=(target_size, target_size), 
                                  mode='bilinear', align_corners=False)
    else:
        lq_resized = self.lq
    
    # 推理...
    
    # 恢复原始尺寸
    if need_resize:
        self.output = F.interpolate(self.output, size=(original_h, original_w), 
                                   mode='bilinear', align_corners=False)
```

### 测试结果

| 数据集 | PSNR | SSIM | 状态 |
|--------|------|------|------|
| UIEB | 25.68 dB | 0.9093 | 成功 |
| LSUI | 22.83 dB | 0.8539 | 成功（resize后） |

## 问题2：本文方法与WF-Diff差距过大

**发现时间**：2026-04-07

### 问题描述

本文方法在UIEB数据集上的测试结果与WF-Diff差距过大：

| 方法 | PSNR | SSIM |
|------|------|------|
| 本文方法 | 14.43 dB | 0.4633 |
| WF-Diff | 25.68 dB | 0.9093 |
| **差距** | **-11.25 dB** | **-0.446** |

### 原因分析

#### 1. 网络架构差距

| 方面 | 本文方法 | WF-Diff |
|------|----------|---------|
| 架构类型 | 简单残差网络 | 双Diffusion模型+UNet |
| 参数量 | ~3.5M | ~100M |
| 感受野 | 小（局部） | 大（全局+局部） |
| 多尺度处理 | 无（单尺度） | 有（多尺度） |
| 细节保留 | 差 | 好（跳跃连接） |

#### 2. 训练策略差距

| 方面 | 本文方法 | WF-Diff |
|------|----------|---------|
| 训练方式 | 分步训练 | 端到端训练 |
| 优化目标 | 各自优化 | 整体优化 |
| 网络配合 | 不配合 | 相互配合 |

#### 3. 训练数据差距

| 方面 | 本文方法 | WF-Diff |
|------|----------|---------|
| 数据集 | UIEB (890对) | LSUI (4000+对) |
| 数据量 | 小 | 大 |

### 解决方案

#### 方案1：端到端训练（优先级高）

**原理**：

```
分步训练：
H网络优化: min L1(H(x), y)      # 只看H网络的输出
去噪网络优化: min L2(D(H(x)), y)  # H固定，只优化D
问题: 两个目标可能冲突

端到端训练：
整体优化: min L(D(H(x)), y)
H和D同时更新，找到整体最优解
```

**优势**：
- H网络知道输出要给去噪网络用
- 去噪网络可以告诉H网络"我需要什么样的输入"
- 两个网络相互配合，整体最优

**预期提升**：+2-3 dB PSNR

#### 方案2：U-Net架构升级（优先级高）

**原理**：

```
当前残差网络：
输入 → 卷积 → 残差块×8 → 卷积 → 输出
问题：感受野小、无多尺度、细节丢失

U-Net架构：
输入 → 编码器(下采样) → 瓶颈层 → 解码器(上采样) → 输出
         ↑_________________↓
              跳跃连接
优势：多尺度特征、大感受野、保留细节
```

**优势**：
- 多尺度特征提取（不同分辨率捕获不同信息）
- 跳跃连接保留细节
- 感受野更大，理解全局信息

**预期提升**：+3-4 dB PSNR

#### 方案3：引入感知损失

```python
# 当前损失函数
loss = L1_loss + 0.1 * SSIM_loss

# 改进损失函数
loss = L1_loss + λ1 * SSIM_loss + λ2 * Perceptual_loss
```

**预期提升**：+1-2 dB PSNR

#### 方案4：扩大训练数据

使用LSUI、EUVP等大规模数据集训练。

**预期提升**：+2-3 dB PSNR

### 解决方案实现

**已创建端到端训练脚本**：`src/train_end_to_end.py`

**核心改进**：

```python
# 原来分步训练
# 步骤1: 训练H网络
h_output = h_net(input)
loss_h = criterion(h_output, target)
loss_h.backward()

# 步骤2: 固定H网络，训练去噪网络
h_output = h_net(input)  # H网络固定
output = denoise_net(h_output)
loss_d = criterion(output, target)
loss_d.backward()

# 现在端到端训练
output, h_output = model(input)  # H网络 + 去噪网络
loss = criterion(output, target)
loss.backward()  # 同时更新两个网络
```

**关键代码**：

```python
class EndToEndModel(nn.Module):
    """端到端模型：H网络 + 去噪网络"""
    
    def __init__(self, h_net, denoise_net):
        super(EndToEndModel, self).__init__()
        self.h_net = h_net
        self.denoise_net = denoise_net
        
    def forward(self, x):
        # H网络处理
        h_output = self.h_net(x)
        # 去噪网络处理
        output = self.denoise_net(h_output)
        return output, h_output

# 优化器同时优化两个网络
optimizer = optim.Adam(model.parameters(), lr=lr)
```
