# 问题记录：WF-Diff显存不足解决方案

## 问题描述

运行WF-Diff测试LSUI数据集时，出现显存不足错误：

```
torch.OutOfMemoryError: CUDA out of memory.
Tried to allocate 1.41 GiB.
GPU 0 has a total capacity of 6.00 GiB of which 0 bytes is free.
```

## 原因分析

| 数据集 | 图像数量 | 图像尺寸 | 显存需求 |
|--------|----------|----------|----------|
| UIEB | 89张 | 256×256 | ~2GB（6GB够用） |
| LSUI | 400张 | 400×300 ~ 1280×1024 | ~4-8GB（6GB不够） |

**根本原因**：Diffusion模型在推理时需要存储大量中间激活值，显存占用与图像尺寸的平方成正比。

## 解决方案

**动态Resize策略**：在推理前将大图像缩小到512×512，推理完成后再恢复原始尺寸。

### 修改的代码

文件：`basicsr/models/Wfdiff_model.py`

**1. feed_data方法 - 记录原始尺寸**

```python
def feed_data(self, data):
    self.pre = data['pre']
    self.lq = data['lq'].to(self.device)
    # 记录原始尺寸，用于推理后恢复
    self.original_size = self.lq.shape[2:]  # (H, W)
    if 'gt' in data:
        self.gt = data['gt'].to(self.device)
```

**2. test方法 - 动态resize推理**

```python
def test(self):
    self.net_g.eval()
    dwt, idwt = DWT(), IWT()
    with torch.no_grad():
        target_size = 512
        original_h, original_w = self.original_size
        
        # 只对大于512的图像进行resize
        need_resize = original_h > target_size or original_w > target_size
        
        if need_resize:
            lq_resized = F.interpolate(self.lq, size=(target_size, target_size), 
                                      mode='bilinear', align_corners=False)
        else:
            lq_resized = self.lq
        
        # 推理
        self.out1, self.out2ll, _, self.out2high, _, self.AA, self.HH = self.net_g(lq_resized, None)
        # ... 中间处理 ...
        
        # 恢复原始尺寸
        if need_resize:
            self.output = F.interpolate(self.output, size=(original_h, original_w), 
                                       mode='bilinear', align_corners=False)
    self.net_g.train()
```

## 为什么这样做？

### 显存占用原理

Diffusion模型的显存占用主要由以下部分组成：

1. **模型权重**：固定开销，约0.37GB（1亿参数 × 4字节）
2. **激活值**：与图像尺寸的平方成正比
3. **中间特征图**：UNet编码器-解码器结构存储大量中间结果

```
显存 ≈ 模型权重 + k × (H × W)²
```

其中k是网络深度相关的系数。

### Resize的好处

| 方面 | 说明 |
|------|------|
| **显存节省** | 512×512比1280×1024节省约80%显存 |
| **兼容性** | 小于512的图像不resize，保持原始质量 |
| **可运行性** | 6GB显存可以运行所有测试 |

### 潜在影响

| 影响 | 说明 |
|------|------|
| **质量损失** | 大图像经过下采样-上采样，可能损失部分细节 |
| **指标变化** | PSNR/SSIM可能略有下降（通常<1dB） |
| **适用场景** | 适合获取SOTA对比基准，不适合追求最高精度 |

## 技术细节

### 为什么用双线性插值？

- `mode='bilinear'`：适合图像上采样/下采样，保持平滑
- `align_corners=False`：保持图像的相对位置关系

### 为什么选择512×512？

- 6GB显存的安全上限
- 大多数水下图像增强模型的训练尺寸
- 平衡显存占用和输出质量

## 总结

通过动态resize策略，在有限显存条件下成功运行WF-Diff测试。这种方法适用于：
- 获取SOTA模型的对比基准
- 显存受限的测试环境
- 快速验证模型效果

对于追求最高精度的场景，建议使用更大显存的GPU或云服务（如Google Colab Pro）。
